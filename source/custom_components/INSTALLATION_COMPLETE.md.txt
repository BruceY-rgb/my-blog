## 🚀 快速开始

### 1. 启用示例组件

**启用时钟组件**：
```bash
# 方法一：使用管理脚本（推荐）
node source/custom_components/manage.js enable clock

# 方法二：手动配置
# 编辑 source/custom_components/components.config.yml
# 设置 clock.enable = true

# 编辑 _config.butterfly.yml
# 在 inject 部分取消注释：
inject:
  head:
    - <link rel="stylesheet" href="/custom_components/clock/clock.css">
  bottom:
    - <script src="/custom_components/clock/clock.js"></script>
```

**启用天气组件**：
```bash
# 1. 获取 API Key
# 访问 https://openweathermap.org/api 注册账户

# 2. 配置 API Key
# 编辑 source/custom_components/weather/weather.js
const WEATHER_CONFIG = {
  apiKey: 'YOUR_API_KEY_HERE',  # 替换为你的 API Key
  defaultCity: 'Beijing',
};

# 3. 启用组件
node source/custom_components/manage.js enable weather
```

### 2. 重新构建网站
```bash
hexo clean && hexo generate
```

### 3. 查看组件状态
```bash
node source/custom_components/manage.js status
```

## ⚠️ 重要注意事项

### 1. 遵循注入原则
✅ **必须做的**：
- 所有自定义 CSS/JS 通过 `_config.butterfly.yml` 中的 `inject` 配置项引入
- 修改 `source/custom_components/` 目录下的文件
- 使用管理脚本或手动编辑配置文件

❌ **严禁做的**：
- 直接修改 `node_modules/hexo-theme-butterfly/` 目录下的任何文件
- 修改主题模板文件（`.pug`、`.ejs` 等）
- 删除 `inject` 配置标记

### 2. 管理脚本标记
在 `_config.butterfly.yml` 中有以下标记，请**不要删除**：
```yaml
# inject:start - 请勿删除此标记，管理脚本将根据此标记自动更新配置
inject:
  head:
    # inject:weather
    # inject:clock
    # inject:end
  bottom:
    # inject:weather
    # inject:clock
    # inject:end
# inject:end - 请勿删除此标记
```

### 3. 组件开发规范

**CSS 规范**：
- 使用 BEM 命名：`cc-component__element--modifier`
- 使用 CSS 变量：`var(--cc-component-color)`
- 遵循移动优先的响应式设计

**JavaScript 规范**：
- 使用 IIFE 避免全局污染
- 所有函数和变量使用 `'use strict'`
- 添加错误处理和日志输出

### 4. 主题升级兼容性

由于所有修改都在用户目录下，升级 Butterfly 主题不会影响您的自定义组件：

```bash
# 升级主题
npm update hexo-theme-butterfly

# 重新应用配置（如果需要）
# 只需重新在 inject 部分添加自定义组件的引用
```
